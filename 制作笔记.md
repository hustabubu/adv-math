## 1 制作放大镜

为了尽量高内聚，将放大镜的代码**封装成一个函数**，方便使用。详情看文件
`src\utils\useMagnifierDom.ts`（已经完成）

三种具体编写方案：
1. **原生操作 DOM 方案**
   - **面向过程**式编写（目前采用）；
   - **面向对象**式编写，也就是封装成**类**，方便使用；
2. 通过 Vue 的**自定义组件** `defineComponent()` 来编写；
3. 通过 Vue 的**自定义元素** `defineCustomElement()` 来编写成 web component.

### 1.1 技术路线

**路线 1： 用 CSS 来制作。**
小、大两张图片，分别是原始、放大。小图背景，大图于放大镜内。
优点：简单、直观。
缺点：不够精细，可扩展性差。

**路线 2：用 canvas 来制作。**
用 canvas 的`clip()` 来需要截取放大镜内的图片内容，再画出来。
优点：精细，可扩展性好。
缺点：需要理解 canvas 的 API，编写代码复杂。

**路线 3：用 SVG 来制作。**
优点：由于是矢量图，放大不变形。
缺点：比 canvas 占一定的资源。

我的需求：需要一定的精度、需要可扩展性、放大的效果好。
故**选择用 Canvas 来制作**。

----

### 1.2 技术难点

**难点 1：** 获取当前页面的图片数据，即要**截图**（无论是路线 1，还是路线 2，都需要截图）。
**实现工具：**
1. html2canvas 库：把 dom 转成 canvas。但库老旧，已经不维护，且效果不理想。
2. modern-screenshoot 库：把 dom 转化为图片，比 html2canvas 好用，但截图也不尽如人意（与页面的原始图有一定的偏差【在**可接受**范围】，但对 grid 布局（或可能是 unocss）不友好，偏差较大【可能是对绝对定位元素有影响，**难以接受**】）。
3. 用原生的 `getDisplayMedia()` 来截图。效果很好！缺点很讨厌，**需要用户授权，没法无感使用**【**难以接受**】。

综上所述，选择 modern-screenshoot 库，减少unocss.

### 2.1 自定义导航栏

在文件 `custom-nav-controls.vue` 中添加的组件，都会在原来导航栏的基础上，往后插入，即插入插槽`<CustomNavControls />`。

目前添加了两个组件：
1. 放大镜组件；
2. 返回目录页。

**细节：**
1. 为了与原来导航栏的风格一致，可以**参考原来内置的组件** `\node_modules\@slidev\client\internals\NavControls.vue` 来制作。
2. ==**Slidev 中的已有的组件的使用**==（包括没有直接暴露出来的）
   - 官方文档没有提及（没有直接暴露出来的），但 Slidev 有的组件，可以类似的通过如下方式导入：
        >`import VerticalDivider from '@slidev/client/internals/VerticalDivider.vue'`
   - 如果官方文档提及有的，
        >在写在幻灯片的 markdown 文件，则直接使用；
        >在自己写的 Vue 文件中，则可以通过 `import *** from '@slidev/client/***.vue'` 来导入。
3. **导航栏的按钮**：
   - **按钮**： `<IconButton>`，其样式通常用 `slidev-icon-btn`
   - 按钮的**扩展菜单**：`<MenuButton>`
   - 垂直分割线：`<VerticalDivider>`
4. **图标使用：** 风格是 Carbon 图标（[Iconify 图标库](https://icon-sets.iconify.design/carbon/?keyword=carbon)），例如
   ```html
    <VerticalDivider />
    <button class="slidev-icon-btn" title="toc" @click="go(2)">
        <carbon:table-of-contents />
    </button>
   ```

## 4 自制 Geogebra 组件 `LuGeogebra`

**目的**：用于展示课件中需要用到的图形和动画。

**定位**：**以展示功能为主**，故不需要提供太多的属性和 API。至于 Geogebra 的制作，最好还是**在外部先制作好，存于 Geogebra 网站（云端）**，**通过 `material_id` 来引入**。

*ps：为了制作的方便（代码着色，代码提示等等辅助），可以自己另外做个 `TypeScript` 项目，在其上制作。*

==**为了一劳永逸，自制的 Geogebra 课件还是应该包含相对完整的功能（以后再慢慢补充），只是在用的时候以展示为主，其它的功能都不调用**==

**技术路线**：通过 [Geogebra 提供的 API](https://geogebra.github.io/docs/reference/en/GeoGebra_Apps_API/)，嵌入用 Vue3 制作的组件内。
- **动态创建** `<script>` 标签加载 `deployggb.js`；
- 结合 Vue 的 `<Suspense>` 和**异步组件**（`defineAsyncComponent`），让 GeoGebra 组件本身也变成**懒加载**的；
- `deployggb.js` 被加载一次后，会被浏览器缓存。后续再初始化其他 GeoGebra 实例时，`window.GGBApplet` 已经存在，`loadGeogebraScript` 函数中的 Promise 会立即解决（`resolve`），**不会重复加载**。

!!! note 一些细节：
    - **TypeScript 类型文件**：把 Geogebra 的 API 说明文档，丢给 DeepSeek 生成，放在组件所在目录下的 `./types`，就可以自动识别。
    - **在 Slidev 中显示的大小问题**：由于 Slidev 的 seriph 主题把内容默认调大（好像是 1.2 倍），导致可能 Geogebra 组件显示不完整，可以**通过 Slidev 内置的 `Transform` 组件适当放缩**。（ps：组件的 `width` 和 `height` 是组件的尺寸，而非在 Slidev 中显示的尺寸）
    简而言之，**通过组件的 `width` 和 `height` 调整图形在 Geogebra 显示的完整性，通过 `Transform` 调整组件在 Slidev 显示的完整性**。

!!! warning 避坑
    **问题一**： 由于**在 Slidev 中**，**组件不会自动卸载**（至少在页面切换后不会自动卸载），则需要一个机制来**自动卸载当前 Geogebra 实例**（否则，大量 Geogebra 的实例在，严重占用内存）
    **解决办法**：
    **方法 1**：增加一个按钮用于调用组件，但需要在每个调用的页面中额外写 JavaScript，**麻烦**；
    **方法 2**：借助于 Slidev 的内置组件 `RenderWhen` 和 `VSwitch` （只有这两个组件有卸载【在 DOM 中移除】功能，而其它的（比如 `VClick`）都只是 显示 | 隐藏） 【**`VSwitch` 通过 属性 `:unmount=true` 来设置点击改变时来卸载前一个插槽**】
    ```html
    <!-- 通过 :unmount=true 来设置点击改变时是否卸载 -->
    <v-switch :unmount=true>
      <template #1> 点击 1 时显示，点击 2 时隐藏。 </template>
      <template #2> 
         <lu-geogebra />
      </template>
      <template #5-7> 点击 5 时显示，点击 7 时隐藏。 </template>
    </v-switch>
    ```
    ```html
    <!-- 通过 context="visible" 来设置 -->
    <RenderWhen context="visible">
      如果可见则渲染内容。
      <lu-geogebra />
    </RenderWhen>
    ```
    **通常情况下**
    - **无脑使用 `RenderWhen`**，
    - **可控使用 `VSwitch`**。
    
    **`VSwitch` 避坑**: 当前 最后一个插槽的点击数字 = 当前页面的最后点击数字 的情况，则**此插槽不会卸载**（其它情况都会卸载），所以最好还是避免这种情况的发生，否则出 bug 时，都找不到原因所在。

!!! tip 性能优化问题
    课件中的链接的图片、Geogebra 组件等等，在**需要用到时才加载到 DOM 中，考虑用 `RenderWhen` 和 `VSwitch`**。
    牺牲一些局部体验的流畅性，提高整体的流畅性。

**原因**：在 Slidev 中，一开始就会把所有的内容都加载到 DOM 中，而实际上是不全需要的（比如：当前讲第一节的内容时，其它章节的内容都是不需要的，但它们都被加载到 DOM 中）。

- 若课件内容少，那就无所谓了；
- 若这里把《高等数学》的所有课件都放进来，再碰上 Github 卡，叠加起来，初次打开就卡到爆。

**目前 Slidev 没有设计按需加载的功能**，只能手动把耗费资源的，先暂时不加载，等需要用时再加载。比如：**图片、Geogebra 组件等等**。

**按需加载问题**：
1. 是否可以在每个章节的 markdown 文件开头和结尾用 `RenderWhen` 标签包裹起来？**不可**，错误提示标签不闭合。
2. 是否和可以在同一个 markdown 文件中，对多个分页用 `RenderWhen` 标签包裹起来？**不可**，错误提示标签不闭合。

3. 看来，`RenderWhen` 只能在课件内，单独的一页之内用（也就是**只能在页内用**），

**如果所有页内都用 `RenderWhen` （相当于以页为单位加载），也有弊端：**
- 额外加标签，量大
- 虽然不提前加载其它章节，但本节课的内容涉及到的页面也没有加载，也不是我想要的。
**我想要的是：以节（或章）为单位，来按需加载。**

**总结**：
**保守方案**：仅就图片、Geogebra 组件等耗资源的用 `RenderWhen`或 `VSwitch`
**折中方案**：逐页来加 `RenderWhen`
**如果保守方案，仍不能缓和卡顿的问题，则用折中方案。**
